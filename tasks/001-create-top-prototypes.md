# 001: トップページ静的プロトタイプ作成

## 目的

stay側とcompany側のトップページを、目的・ユーザー・CTAが混ざらない形で静的HTML/CSSとして作成する。最終的なWordPress/SWELL化の前に、構成、導線、トーンを確認できる状態にする。

## stay側トップページ要件

### 対象ユーザー

- 北海道で宿泊先を探している旅行者
- シマエナガニセコ、シマエナガ札幌を比較したい人
- 写真、設備、アクセス、周辺情報を確認してから予約したい人

### 必須セクション

1. ヒーロー
   - 北海道、雪、森、静けさを感じる写真中心の構成
   - 主CTA: Beds24で空室を確認
   - 補助CTA: 施設を見る

2. Beds24空室検索エリア
   - 仮UIとして設置
   - `data-beds24-replace="stay-availability-widget"`を付ける
   - WordPress化時にBeds24公式プラグインまたは埋め込みコードへ差し替える

3. 施設一覧
   - シマエナガニセコ
   - シマエナガ札幌
   - 各カードに写真、短い説明、Beds24予約ボタン、施設詳細ボタンを置く

4. 宿泊前の安心材料
   - 清潔感
   - 設備
   - アクセス
   - FAQへの導線

### 禁止事項

- 民泊運営相談を主CTAにしない
- 予約機能を自作しない
- 決済、在庫、料金計算を実装しない

## company側トップページ要件

### 対象ユーザー

- 民泊運営を委託したい物件オーナー
- 民泊立ち上げを検討している個人・法人
- Beds24導入やOTA運用を相談したい人

### 必須セクション

1. ヒーロー
   - 信頼感、法人感、相談しやすさを出す
   - 主CTA: 民泊運営を相談する
   - 補助CTA: 支援内容を見る

2. 支援メニュー
   - 民泊運営代行
   - 民泊立ち上げ支援
   - Beds24導入・OTA運用支援

3. 信頼材料
   - 運営ノウハウ
   - 北海道エリア理解
   - 改善提案
   - 実績掲載エリア

4. 相談の流れ
   - 現状確認
   - 課題整理
   - 支援方針

5. 問い合わせフォーム仮UI
   - `data-form-replace="company-contact-form"`を付ける
   - WordPress化時にフォームプラグインまたは外部フォームへ差し替える

### 禁止事項

- 宿泊予約CTAを置かない
- Beds24予約ボタンを置かない
- 宿泊者向けの写真・文言を主導線にしない
