# DESIGN.md

このファイルは、AIエージェントと実装者がUIの方向性を揃えるためのデザイン仕様です。awesome-design-md-jpの考え方を参考に、日本語UI、余白、行間、CJKフォント、禁則処理、トーンを明示します。

## 共通原則

- スマホファーストで設計する
- 日本語本文は詰めすぎず、行間を広く取る
- 本文の最大幅は読みやすさを優先し、長文を横に伸ばしすぎない
- CTAは1画面内で迷わず見つかる位置に置く
- 角丸は原則8px以下
- カードを多用しすぎず、必要な比較・一覧に限定する
- シマエナガはブランドの象徴として控えめに扱う
- かわいさより、清潔感・静けさ・信頼感を優先する
- 写真は装飾ではなく、施設・場所・状態を判断できる情報として使う

## 日本語タイポグラフィ

### フォント

```css
font-family:
  "Noto Sans JP",
  "Hiragino Sans",
  "Yu Gothic",
  "YuGothic",
  "Meiryo",
  system-ui,
  sans-serif;
```

見出しに明朝系を使う場合は、限定的に以下を使う。

```css
font-family:
  "Noto Serif JP",
  "Yu Mincho",
  "Hiragino Mincho ProN",
  serif;
```

### 組版

- 本文行間: `1.75`から`1.9`
- 見出し行間: `1.25`から`1.4`
- 本文文字サイズ: モバイル`15px`から`16px`、デスクトップ`16px`から`17px`
- キャプション: `12px`から`13px`
- 英字ラベルは大文字にしすぎず、letter-spacingは`0.08em`から`0.14em`
- 日本語本文のletter-spacingは原則`0`
- 長い日本語は自然に折り返す
- CSSでは`line-break: strict;`、`overflow-wrap: anywhere;`を必要箇所に使う
- OpenTypeは必要に応じて`font-feature-settings: "palt" 1, "kern" 1;`

## stay側デザイン方針

### 印象

- 北海道、雪、森、静けさ、清潔感
- 宿泊体験を想像できる写真中心
- 予約までの導線は短く、迷わせない
- 旅前の不安を解消する落ち着いたUI

### カラー

| Token | Hex | 用途 |
| --- | --- | --- |
| `--stay-paper` | `#fffdfa` | 背景の白 |
| `--stay-snow` | `#f7f6f2` | セクション背景 |
| `--stay-frost` | `#dfe9e7` | 雪・空気感 |
| `--stay-cedar` | `#4f6255` | 予約CTA、見出しアクセント |
| `--stay-wood` | `#9a7656` | 補助アクセント |
| `--stay-ink` | `#252522` | 本文・見出し |
| `--stay-muted` | `#6d6a60` | 補助テキスト |

### 余白

- モバイル左右余白: `20px`
- セクション上下余白: `64px`以上
- デスクトップ最大幅: `1120px`
- 写真とテキストの間隔: `24px`以上
- 予約CTA周辺はタップしやすい余白を確保する

### 写真

- ヒーロー: 北海道らしい雪、森、宿の空気感を伝える写真
- 施設ページ: 室内、寝具、水回り、設備、外観、周辺の順に判断材料を優先
- 暗い写真、過度なぼかし、抽象的な写真だけの構成は避ける
- 画像には内容が分かるaltを入れる

### CTA

- 主CTA: `Beds24で予約`
- 補助CTA: `空室を確認`, `施設詳細を見る`
- 予約機能は自作しない
- Beds24のボタン、空室検索ウィジェット、外部予約ページ導線を差し替えられる仮UIにする

## company側デザイン方針

### 印象

- 信頼感、法人感、運営ノウハウ
- 相談しやすいが、軽く見せすぎない
- 事業内容、対応範囲、実績が整理されている
- 宿泊予約サイトより情報密度を少し高める

### カラー

| Token | Hex | 用途 |
| --- | --- | --- |
| `--company-paper` | `#fffdfa` | 背景 |
| `--company-mist` | `#eef1ed` | セクション背景 |
| `--company-ink` | `#20211f` | 本文・CTA |
| `--company-cedar` | `#394d42` | 信頼感の主色 |
| `--company-steel` | `#68716b` | 補助テキスト |
| `--company-wood` | `#8f6f51` | 実績・番号アクセント |
| `--company-line` | `#ded9cf` | 境界線 |

### 余白

- モバイル左右余白: `20px`
- セクション上下余白: `56px`から`88px`
- 情報カード間隔: `16px`から`24px`
- 一覧や比較は余白を取りつつ、同一視界で比較できる密度にする

### 写真・図版

- 実績、運営現場、清掃、設備、管理画面の雰囲気を伝える写真を優先
- 抽象的なビジネス写真だけで構成しない
- 必要に応じてプロセス図、チェックリスト、対応範囲表を使う

### CTA

- 主CTA: `民泊運営を相談する`
- 補助CTA: `支援内容を見る`, `会社概要を見る`
- 宿泊予約CTAは置かない
- Beds24は予約導線ではなく、導入・運用支援サービスとして扱う

## コンポーネント方針

### ヘッダー

- stay側: 施設ページ、空室確認、FAQへの導線を優先
- company側: 支援内容、実績、会社概要、問い合わせへの導線を優先
- モバイルではリンク数を絞り、主CTAを目立たせる

### カード

- 施設カード、支援メニュー、実績、FAQに限定して使う
- カード内の見出しは短く、本文は3行程度までを目安にする
- カードの中にカードを入れない

### フォーム仮UI

- 静的プロトタイプでは送信処理を作らない
- 入力項目の見た目だけを配置し、`data-form-replace`を付ける
- WordPress化時にフォームプラグインまたは外部フォームへ差し替える

### Beds24仮UI

- `data-beds24-replace`を付ける
- WordPress化時にBeds24公式プラグイン、埋め込みコード、または外部予約ページリンクへ差し替える
- 在庫、料金、決済、予約確定のロジックはサイト内に実装しない

## アクセシビリティ

- CTAのタップ領域は44px以上
- テキストと背景のコントラストを確保する
- 背景画像の上に長文を置かない
- `alt`は画像の役割に合わせて設定する
- フォーカス状態を消さない
- 色だけで状態を伝えない

## WordPress / SWELL移植メモ

- 静的プロトタイプでは、後でSWELLの固定ページ・ブログパーツへ分解しやすいようにセクション単位で構造化する
- `data-wp-section`を付け、移植時のブロック単位を明確にする
- 繰り返し項目はSWELLのリッチカラム、関連記事、ブログパーツ、カスタムHTMLへ置き換えやすい粒度にする
- CSSはグローバルに広げすぎず、サイト別のクラス接頭辞またはセクションスコープを使う
