# AGENTS.md

このプロジェクトでCodexが守るべき実装ルール。

## 基本方針

- 最初からWordPressテーマ化しない
- まずは静的HTML/CSSプロトタイプを作る
- 後でWordPress/SWELLへ移植しやすい構造にする
- 宿泊予約サイトと民泊相談サイトを混ぜない
- stay側とcompany側でユーザー、目的、CTA、トーンを分ける

## サイト分離

- stay側は宿泊者向けに限定する
  - 施設紹介
  - 写真
  - 設備
  - アクセス
  - 周辺観光
  - FAQ
  - Beds24予約導線
- company側は物件オーナー向けに限定する
  - 民泊運営代行
  - 民泊立ち上げ支援
  - Beds24導入・OTA運用支援
  - 実績
  - 会社概要
  - 問い合わせ

## WordPress / SWELL化を前提にした実装方針

- 静的HTMLではセクションごとに`data-wp-section`を付ける
- 後でSWELLの固定ページ、ブログパーツ、カスタムHTML、リッチカラムへ移しやすい粒度にする
- セクション名、クラス名、コメントは実装意図が分かるようにする
- CSSは共通トークン、stay用、company用を分ける
- ページ固有の見た目はスコープを限定する
- 画像、CTA、フォーム、Beds24エリアは差し替えやすいHTMLにする
- PHPテンプレート、functions.php、WordPressテーマファイルは、依頼があるまで作らない

## Beds24連携ルール

- 予約システムは自作しない
- 在庫管理、料金計算、予約確定、決済処理は実装しない
- Beds24連携部分は仮UIとして作る
- 仮UIには`data-beds24-replace`を付ける
- 空室検索ウィジェット設置エリアは`role="region"`と分かりやすい`aria-label`を付ける
- WordPress化時にBeds24公式プラグイン、Beds24埋め込みコード、または外部予約ページリンクへ差し替える

## フォーム・問い合わせルール

- 静的プロトタイプでは送信処理を作らない
- company側の問い合わせフォームは仮UIとして作る
- 仮UIには`data-form-replace`を付ける
- WordPress化時にフォームプラグインまたは外部フォームへ差し替える

## デザイン実装ルール

- DESIGN.mdを優先して参照する
- 日本語UIとして読みやすい余白、行間、文字サイズを守る
- スマホファーストで作る
- カードの角丸は8px以下
- カードの中にカードを入れない
- 装飾だけのグラデーション、ぼかし、抽象背景に頼らない
- 写真は内容が分かるものを使う
- シマエナガ要素は控えめにする

## リンク・CTAルール

- stay側の主CTAはBeds24予約導線
- company側の主CTAは問い合わせ・相談導線
- stay側にcompany側の主CTAを混ぜない
- company側に宿泊予約CTAを混ぜない
- 仮リンクは、後で差し替える目的が分かる属性またはコメントを付ける

## 検証ルール

- 変更後は`git diff --check`を実行する
- 静的HTMLはローカルHTTPまたはファイル確認でリンクと表示を確認する
- 主要ページのHTTP 200を確認する
- 公開やpushはユーザーが明示した場合に行う
